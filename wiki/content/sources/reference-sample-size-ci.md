# TFT 数据分析: 样本量、置信区间与偏差

来源: [Reddit - morbrid (MetaTFT dev)](https://www.reddit.com/r/CompetitiveTFT/comments/1kwpx7w/)

## 核心概念

### 1. 样本量 (Sample Size)
- 低样本量 (<300 games): 数据不可靠，少数 highroll/lowroll 会严重偏移均值
- 高样本量: 均值更接近真实值，随机波动被稀释

### 2. 置信区间 (Confidence Intervals)
- 不是一个均值，而是一个**范围**
- 例: 均名 4.3, 95% CI [4.2, 4.4] → 真实均名有 95% 概率在 4.2~4.4 之间
- **宽区间 = 不确定性高**（数据少）
- **窄区间 = 确定性高**（数据多）

### 3. "Worst Case" 排序
- 按置信区间上界排序（对均名来说上界 = 最差情况）
- 均名 4.0 + 上界 4.1 的装备 > 均名 3.8 + 上界 4.3 的装备（前者更稳定可靠）

### 4. 多重比较问题 (Multiple Comparisons Problem)
- 表格有 20+ 行时，至少有 1 行纯靠随机就会看起来特别好/差
- **对策: 行数越多，用更高的置信水平**（如 50+ 行用 99% 而非 95%）

## 数据偏差 (Biases) — 置信区间无法修复的

### 幸存者偏差 (Survivorship Bias) ⚠️ 最重要
- 后期才拿到的装备（如晚期选秀）统计上看起来很强
- 原因: 你能活到后期 = 你本身就强，不是装备让你强
- **副C装备** 尤其容易受此影响（通常是第二个被装备的棋子）

### 玩家行为偏差 (Player Behavior Bias)
- 玩家倾向于出"公认好"的装备，会攒组件等最优装
- 早合次优装的人更少 → 热门装备可能看起来更差（因为包含了攒组件流血的人）

### 采样偏差 (Sampling Bias)
- 添加筛选条件本身会引入偏差
- 例: 筛选"3星3费" → 数据天然好看，因为排除了没三星的人（那些人已经输了）
- 例: 筛选"2星4费" → 同理

### Place Change/Delta 的陷阱
- "有 vs 无" 对比时，如果某装备 95% 场次都出，那"无"的样本量反而很小
- 两个均值的误差会叠加

## Takeaways
1. 样本量 < 300 → 持保留态度，看置信区间
2. 评估小众装备/神器时用 Advanced Mode
3. "Worst Case" 排序找稳定选择
4. 时刻考虑幸存者偏差等统计陷阱
